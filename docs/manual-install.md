# Manual install (managed mode, no installer script)

Every step `kiosk-setup.sh` performs, by hand. Useful for understanding the
system or debugging a broken install. Assumptions: fresh Ubuntu Server 24.04,
kiosk user **`viewer`** (substitute yours throughout), the repo's `agent/`
folder copied to the box at `/tmp/agent/`, and the **manager URL** +
**enrollment token** from the manager's Settings page.

## 1. Kiosk user and packages

```bash
sudo adduser viewer
sudo apt-get update
sudo apt-get install -y --no-install-recommends \
  xorg xinit x11-xserver-utils xserver-xorg-legacy mpv i965-va-driver vainfo \
  openssh-server curl ca-certificates jq wpasupplicant
sudo usermod -aG render,video viewer
```

## 2. Allow X to start from a systemd service

Overwrite `/etc/X11/Xwrapper.config`:

```
allowed_users=anybody
needs_root_rights=yes
```

## 3. Kiosk state directory

```bash
sudo mkdir -p /etc/kiosk
printf 'KIOSK_USER=viewer\nKIOSK_HOME=/home/viewer\n' | sudo tee /etc/kiosk/kiosk.conf
```

## 4. Display loop and display service

```bash
sudo install -m 750 -o root -g viewer /tmp/agent/kiosk-cams /etc/kiosk/cams.sh
sudo install -m 644 /tmp/agent/kiosk-display.service /etc/systemd/system/kiosk-display.service
sudo sed -i 's/@KIOSK_USER@/viewer/' /etc/systemd/system/kiosk-display.service
```

`cams.sh` re-reads `/etc/kiosk/config.json` on every mpv respawn. Until the
manager sends config it logs "waiting for /etc/kiosk/config.json" and shows a
black screen - expected.

## 5. Agent, trigger, agent service

```bash
sudo install -m 755 /tmp/agent/kiosk-agent /usr/local/sbin/kiosk-agent
sudo install -m 755 /tmp/agent/kiosk-trigger /usr/local/sbin/kiosk-trigger
sudo install -m 644 /tmp/agent/kiosk-agent.service /etc/systemd/system/kiosk-agent.service
```

## 6. Manager connection details

```bash
sudo tee /etc/kiosk/agent.env > /dev/null <<'EOF'
MANAGER_URL=http://YOUR-MANAGER:5000
ENROLL_TOKEN=token-from-settings-page
EOF
sudo chmod 600 /etc/kiosk/agent.env
```

The agent appends `API_TOKEN=` and `SCREEN_ID=` after registering.

## 7. Sudoers rule for instant sync

The manager SSHes in as the kiosk user and may run exactly one command as
root. Always validate with visudo before installing:

```bash
cat > /tmp/kiosk-trigger.sudoers <<'EOF'
Defaults!/usr/local/sbin/kiosk-trigger env_keep += "SSH_ORIGINAL_COMMAND"
viewer ALL=(root) NOPASSWD: /usr/local/sbin/kiosk-trigger
EOF
sudo visudo -cf /tmp/kiosk-trigger.sudoers && \
  sudo install -m 440 -o root -g root /tmp/kiosk-trigger.sudoers /etc/sudoers.d/kiosk-trigger
```

## 8. WiFi helper

```bash
sudo install -m 755 /tmp/agent/kiosk-apply-wifi /usr/local/sbin/kiosk-apply-wifi
```

WiFi stays permanently configured at a worse route metric than wired, so a
cabled screen always prefers the cable and falls over to WiFi the moment it's
gone; a WiFi-only screen just uses WiFi. The manager distributes the fleet-wide
SSID/password (Settings page) and the agent applies it on sync — so normally
there is nothing more to do here.

Only for a screen that must reach the manager over WiFi *before* its first
sync (e.g. installed somewhere with no cable), set it once by hand:

```bash
sudo kiosk-apply-wifi "YourSSID" "your-password"   # --clear to remove
```

## 9. SSH key updater

Generated by the installer, so create it by hand. `URL` stays empty in
managed mode (the agent passes a key file fetched from the manager):

```bash
sudo tee /usr/local/sbin/kiosk-update-ssh-keys > /dev/null <<'EOF'
#!/bin/bash
# Rewrites the managed block in the kiosk user's authorized_keys.
# Usage: kiosk-update-ssh-keys [keyfile]   (no arg: fetch from $URL)
# If the source yields no valid keys, authorized_keys is left unchanged.
set -euo pipefail
URL=""
KEY_USER="viewer"
AK_DIR="/home/viewer/.ssh"
AK="$AK_DIR/authorized_keys"
BEGIN="# >>> kiosk-managed-keys (do not edit between markers) >>>"
END="# <<< kiosk-managed-keys <<<"

TMP=$(mktemp)
trap 'rm -f "$TMP" "$TMP.keys" "$TMP.new"' EXIT

if [[ -n ${1:-} ]]; then
  cp "$1" "$TMP"
elif [[ -n $URL ]]; then
  curl -fsSL --max-time 30 "$URL" -o "$TMP"
else
  echo "kiosk-update-ssh-keys: no key file argument and no URL configured" >&2
  exit 1
fi

grep -E '^(ssh|ecdsa|sk)-[A-Za-z0-9@.-]+ [A-Za-z0-9+/=]+' "$TMP" > "$TMP.keys" || true
if [[ ! -s "$TMP.keys" ]]; then
  echo "kiosk-update-ssh-keys: no valid keys in source - authorized_keys left unchanged" >&2
  exit 1
fi

install -d -m 700 -o "$KEY_USER" -g "$KEY_USER" "$AK_DIR"
touch "$AK"
awk -v b="$BEGIN" -v e="$END" '$0==b{skip=1} !skip{print} $0==e{skip=0}' "$AK" > "$TMP.new"
{ echo "$BEGIN"; cat "$TMP.keys"; echo "$END"; } >> "$TMP.new"
install -m 600 -o "$KEY_USER" -g "$KEY_USER" "$TMP.new" "$AK"
echo "kiosk-update-ssh-keys: installed $(wc -l < "$TMP.keys") key(s) for $KEY_USER"
EOF
sudo chmod 755 /usr/local/sbin/kiosk-update-ssh-keys
```

## 10. Start everything

```bash
sudo systemctl daemon-reload
sudo systemctl disable getty@tty1.service
sudo systemctl enable kiosk-agent kiosk-display
sudo systemctl restart kiosk-agent kiosk-display
```

## 11. Verify

`journalctl -u kiosk-agent -f` should show, within ~15 s:
`register: enrolled as <hostname>`, then `poll: ok (...)` heartbeats, then
`manager-key: installed v1 for viewer (instant sync enabled)` and
`ssh-keys: applied vN`. The screen appears on the manager's Screens page as
"needs configuration"; configure layout/NVR there and video starts a few
seconds after saving. If a fleet WiFi network is set in the manager, the first
config sync also logs `kiosk-apply-wifi: WiFi configured for "..."`.

Optional hardware decode check:
`LIBVA_DRIVER_NAME=i965 vainfo --display drm --device /dev/dri/renderD128 | grep VAProfileH264`

Hardware reminders: BIOS "After Power Failure" = Power On; NVR substreams
set to H.264 with identical resolutions across channels.

## Upgrading an old (pre-manager) autologin install by hand

```bash
sudo rm -f /etc/systemd/system/getty@tty1.service.d/autologin.conf
sudo rmdir --ignore-fail-on-non-empty /etc/systemd/system/getty@tty1.service.d
sudo sed -i '/# kiosk-autostart/,/^fi$/d' /home/viewer/.bash_profile
sudo rm -f /usr/local/sbin/kiosk-render-cams   # retired generated-cams design
```

Then follow the normal steps above.
